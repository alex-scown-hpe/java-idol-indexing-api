225     /**
226      * This parameter is identical to the <strong>KillDuplicates</strong> parameter of the <strong>DREADD</strong> 
227      * indexing command.
228      * <p />
229      * You can enter one of the following options to specify how IDOL server handles duplicate text when it adds 
230      * documents to the target IDOL server. Note that if you postfix any of these options with <strong>=2</strong>, the 
231      * KillDuplicates process is applied to all IDOL server databases (rather than just the database into which the 
232      * current IDX or XML file is being indexed):
233      * <dl>
234      *   <dt><strong>NONE</strong></dt>
235      *   <dd>Documents in IDOL server are never replaced with new documents.</dd>
236      *   <dt><strong>REFERENCE</strong></dt>
237      *   <dd>If the document being indexed has the same <strong>DREREFERENCE</strong> field value as a document that 
238      *       already exists in IDOL server, IDOL server deletes the existing document and replaces it with the new 
239      *       document.</dd>
240      *   <dt><strong>REFERENCEMATCH<i>N</i></strong></dt>
241      *   <dd>If the content of the document being indexed is more than <strong><i>N</i></strong> percent similar to the
242      *       content of a document that already exists in the IDOL server database, IDOL server deletes the existing 
243      *       document and replaces it with the new document.</dd>
244      *   <dt><strong><i>FieldName</i></strong></dt>
245      *   <dd>If the document being indexed contains a <strong><i>FieldName</i></strong> Reference field with the same 
246      *       value as the <strong><i>FieldName</i></strong> Reference field in a document that already exists in IDOL 
247      *       server, IDOL server deletes the existing document and replaces it with the new document.</dd>
248      * <dl>
249      * To specify multiple Reference fields, separate the fields with a plus sign, a space, or an underscore. IDOL 
250      * server deletes documents containing any of the specified fields with identical content. To apply the 
251      * KillDuplicates process to multiple fields containing underscores, add <strong>&v4</strong> to the command. This 
252      * restricts the separators to a plus sign, a space, and a comma only.
253      * <p />
254      * <strong>Note:</strong> fields are identified as Reference fields through field processes in the IDOL server 
255      * configuration file (see <strong>Reference fields</strong> in the IDOL server manual). If you use a <strong><i>
256      * FieldName</i></strong> Reference field to eliminate duplicate documents, IDOL server automatically reads any 
257      * fields that are listed alongside this field for the <strong>PropertyFieldCSVs</strong> parameter in the field 
258      * process, and also uses these fields to eliminate duplicate documents. If you want to define multiple reference 
259      * fields but don’t want them all to be used for document elimination, you need to set up multiple field processes 
260      * (see <strong>Using Reference fields to eliminate duplicate copies of documents during indexing</strong> in the 
261      * IDOL server manual).
262      * <p />
263      * If you do not set <strong>KillDuplicates</strong>, it defaults to the option specified for <strong>KillDuplicates
264      * </strong> in the IDOL server configuration file’s <strong>[Server]</strong> section.
265      * 
266      * @param killDuplicates How to handle duplicates when adding documents to the target IDOL server.
267      */
268     public void setKillDuplicates(final String killDuplicates) {
269         put(PARAM_KILL_DUPLICATES, killDuplicates);
270     }
